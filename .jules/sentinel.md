## 2024-11-06 - Missing Permission Checks in File System System Calls
**Vulnerability:** File system system calls (`sys_open`, `sys_mkdir`, `sys_unlink`) did not verify if the calling user task possessed the appropriate access permissions (read/write) for the target nodes or parent directories, allowing any process to arbitrarily manipulate files regardless of their POSIX ownership.
**Learning:** In a monolithic kernel with VFS, discovering the `fs_node_t` via `vfs_lookup` is not sufficient for secure operation. System calls must enforce POSIX-style bitmask checks explicitly at the edge boundary before proceeding with mutations or returning handles to user-space.
**Prevention:** Implement and reuse a central permission validation utility (e.g., `check_node_permission`) against the task's active `uid`/`gid` in all VFS wrapper system calls prior to invoking lower-level filesystem drivers.
